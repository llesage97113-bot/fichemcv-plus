# FicheMCV+ — Manifeste du dossier d’antériorité

## 1. Objet du dossier

Ce dossier vise à constituer une preuve structurée de l’existence, de la conception, de l’architecture, des règles métier et de l’état de développement du projet FicheMCV+.

Il a pour objectif de préparer un futur dépôt e-Soleau auprès de l’INPI ou tout autre dispositif permettant d’établir une preuve d’antériorité.

## 2. Projet concerné

**Nom du projet :** FicheMCV+  
**Porteur du projet :** Laurent Lesage  
**Nature :** application web pédagogique de suivi, rédaction, accompagnement et validation des fiches CCF pour le Bac Pro Métiers du Commerce et de la Vente.

## 3. Contenu du dossier

Le dossier comprend notamment :

- une description générale du projet ;
- un cahier des charges fonctionnel ;
- une architecture technique ;
- les règles métier principales ;
- la roadmap ;
- les preuves Git ;
- un plan de captures d’écran ;
- les captures d’écran produites ;
- les références aux extraits de code significatifs.

## 4. Structure des fichiers

- `01_description_du_projet.md`
- `02_cahier_des_charges_fonctionnel.md`
- `03_architecture_technique.md`
- `04_regles_metier.md`
- `05_roadmap.md`
- `06_preuves_git.md`
- `07_captures_ecran.md`
- `08_extraits_code_significatifs.md`
- `git_history.txt`
- `captures_ecran/`

## 5. Finalité

Ce dossier ne constitue pas en lui-même un dépôt de marque ni un brevet.

Il sert à documenter et dater :

- le concept ;
- les choix fonctionnels ;
- les règles métier ;
- les interfaces ;
- l’architecture ;
- l’état d’avancement du projet ;
- les preuves techniques disponibles.

## 6. Usage futur

Ce dossier pourra servir à :

- préparer une e-Soleau ;
- présenter le projet à un tiers ;
- documenter l’antériorité de FicheMCV+ ;
- soutenir une réflexion sur un dépôt de marque ;
- préparer une diffusion ou commercialisation future.

## 7. Points de vigilance

Certaines captures peuvent contenir des données personnelles ou scolaires.

Avant tout dépôt externe ou partage avec un tiers, il conviendra de vérifier que les éléments sensibles sont anonymisés ou masqués :

- noms d’élèves ;
- emails techniques ;
- mots de passe provisoires ;
- numéros candidats ;
- données personnelles.

## 8. Synthèse

Le présent dossier constitue une première base documentaire solide pour établir l’antériorité du projet FicheMCV+ et préparer sa protection progressive.
